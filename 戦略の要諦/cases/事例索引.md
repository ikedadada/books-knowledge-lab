# 事例索引

本書の事例は、一般法則の証明ではなく、定義・原則・失敗パターンを説明する材料として扱う。

| 事例 | 本書での論点 |
|---|---|
| Netflix | 戦略は固定した長期目標ではなく、次々に現れる課題への対応である |
| クエストコー | 前向きな標語から、困難だが攻略可能な顧客課題へ焦点を移す |
| iPhone | 自社ではなく顧客の未解決課題を診断し、誤った類推を避ける |
| エアランド・バトル | 問題のフレームと組織の戦い方を同時に変える |
| ペツル、サウスウエスト航空、イケア | 製品・顧客・活動を絞ることで行動の一貫性を作る |
| スペースシャトル | 利害関係者の合議と非現実的な見積もりが設計の一貫性を壊す |
| IBM再生 | 組織の歴史に根ざした慣性を診断し、組織能力を組み替える |
| Appleとスティーブ・ジョブズ | 財務指標ではなく製品、設計、事業上の選択へ集中する |
| オペレーション・ワープスピード | 抽象的な将来計画より、具体的な障害を除く一貫した行動が成果を生む |

事例を別の状況へ適用するときは、表面的な類似ではなく、結果を生んだ因果関係と条件を比較する。
